UML Class Diagram - Factory Method Pattern

FC Cakrawala Football Club System

Factory Method Pattern Implementation

This UML diagram shows the improved design using the Factory Method pattern. The key changes include:

Key Changes Made:

  1. Abstract PersonFactory: Defines the interface for creating Person objects
  2. Concrete Factories: CoachFactory, PlayerFactory, and StaffFactory implement the factory interface
  3. Abstract Person Class: Base class for all person types
  4. Encapsulation: Object creation logic is encapsulated in factory classes
  5. Extensibility: Easy to add new person types by creating new factories

Benefits of Factory Method Pattern:

  • Decouples object creation from the client code
  • Makes the code more flexible and maintainable
  • Follows the Open/Closed Principle (open for extension, closed for modification)
  • Provides a consistent interface for creating different types of persons
